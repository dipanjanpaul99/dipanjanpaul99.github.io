# Technical Architecture  
Implementation details:
- **Framework:** Next.js App Router + React + TypeScript
- **Styling:** Tailwind CSS v4 via `@theme inline` in `app/globals.css`
- **Components:** shadcn/ui primitives in `components/ui`
- **Icons:** Lucide React
- **Fonts:** IBM Plex Sans for UI, Fraunces for display headings, IBM Plex Mono for technical labels (via `next/font`)
- **Dark mode:** `next-themes`, class-based, dark default
- **Utilities:** `cn()` from `@/lib/utils`
- **Deployment:** GitHub Pages via `output: 'export'` in `next.config.ts`
- **Async APIs:** Use `await` for `cookies()`, `headers()`, `params`, `searchParams` (Next.js 15+)
- **Data pattern:** Static data objects in `data/` directory, imported by server components

# Design System (see DESIGN.md)
Apply the **"founder war-room / blueprint desk"** visual system:

- **Background:** Dark ink (`oklch(0.15 0.038 252)`) with layered blueprint grid texture (`.planner-bg`)
- **Primary (commands):** Amber (`oklch(0.78 0.16 61)`)
- **Accent (highlights):** Teal (`oklch(0.77 0.134 178)`)
- **Depth:** Violet glow accents
- **Panels:** Glassy translucent surfaces (`.glass-panel`, `.paper-card`)
- **Grid surface:** `.blueprint-surface` for compact areas
- **Labels:** `.micro-label` ‚Äî mono, uppercase, wide tracking

Use semantic tokens only (`bg-background`, `text-muted-foreground`, etc.) ‚Äî never raw color values.

# Components
All UI components use shadcn/ui with these rules (see `.agents/skills/shadcn/SKILL.md`):
- Layout via `flex gap-*` ‚Äî never `space-x-*` / `space-y-*`
- Equal dimensions via `size-*` ‚Äî never `w-* h-*`
- Conditional classes via `cn()` ‚Äî never template literal ternaries
- Icons in buttons use `data-icon="inline-start"` / `data-icon="inline-end"`
- Full Card composition (`CardHeader`/`CardTitle`/`CardContent`/`CardFooter`)
- Semantic variants (`variant="outline"`, `size="sm"`) before custom styles
